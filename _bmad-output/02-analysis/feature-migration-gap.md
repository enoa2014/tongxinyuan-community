# 新旧网站功能对比与迁移清单 (Feature Gap Analysis)

基于对旧版网站代码的深度拆解与新版产品设计文档 (`product-design.md`) 的对比分析。

## 1. 核心功能对比矩阵

| 功能模块 | 旧版网站 (Legacy) | 新版网站 (New 2026) | 状态 | 备注 |
| :--- | :--- | :--- | :--- | :--- |
| **内容管理 (CMS)** | 新闻、关于、项目点、招募信息 | 首页、服务中心、透明度报告、资源库 | ✅ **增强** | 新版结构更清晰，增加了“资源库”和“透明度披露”。 |
| **用户身份** | 通用会员 (Member) | **分角色体系** (家庭/志愿者/捐赠人) | 🚀 **升级** | 以前是单一身份，现在根据角色提供不同服务。 |
| **捐赠功能** | 仅展示银行账号 (静态页) | **在线月捐/支付** (交互) | 🚀 **升级** | 从信息展示升级为交易转化。 |
| **志愿者功能** | 仅展示招募文章 (静态页) | **任务大厅/排班/签到** (交互) | 🚀 **升级** | 从被动阅读升级为主动参与。 |
| **社区互动** | **论坛/话题 (Topics)** | 🚫 **无** (未规划) | ⚠️ **缺失** | 旧版允许用户发帖，新版目前没有规划BBS功能。 |
| **评论系统** | **全局文章评论** | 🚫 **无** (未规划) | ⚠️ **缺失** | 新版资讯页未提及评论功能。 |
| **服务申请** | 线下/电话 | **在线建档/预约** (厨房/住宿) | ✨ **新增** | 核心业务数字化。 |

## 2. 需“复活”的旧版功能清单 (Migration List)

以下是旧版存在但新版规划中**缺失或未明确**的功能，建议根据优先级加入新版：

### 🔥 P1：必须迁移 (Must Have)
这些是核心资产，丢失会造成断层。

1.  **项目点详情页 (Locations)**
    *   *旧版*: 包含万秀、北湖、兴宁三个服务点的详细地址、联系人和环境图。
    *   *新版*: 目前合并在“生活支持”中。
    *   *建议*: 保留“服务网络”独立页面，方便用户按地图查找。

2.  **历史文章数据 (Article Archives)**
    *   *旧版*: 包含多年的机构动态和媒体报道。
    *   *建议*: 将 `txy_articles` 表数据清洗后导入新 CMS，保留历史沉淀。

3.  **老用户账号关联 (User Migration)**
    *   *旧版*: `txy_members` 表存储了用户的微信 OpenID。
    *   *建议*: 新版上线时，允许老用户通过微信一键登录“找回”身份（即使以前功能很简单，但这是情感连接）。

### ⚠️ P2：需决策是否保留 (Decide)
这些功能在旧版存在，但可能与新版“工具化”定位冲突，带来了运营/审核成本。

4.  **社区论坛/话题 (User Topics)**
    *   *旧版*: 允许用户发布“话题” (`txy_topics`)，类似贴吧。
    *   *分析*: 这是一个**高风险、高运营成本**的功能（需审核内容）。且目前微信群通常替代了BBS的功能。
    *   *建议*: **暂不迁移**。用“家长互助会”（线下活动+线上报名）或“故事分享”（投稿制）替代不可控的灌水论坛。

5.  **文章评论 (Comments)**
    *   *旧版*: 允许对新闻/话题评论。
    *   *分析*: 同样涉及内容审核。对于官网资讯，评论区的活跃度通常很低且易招致垃圾广告。
    *   *建议*: **移除**。将互动引导至微信公众号后台，官网保持清爽专业。

## 3. 建议的修正路线图

基于上述分析，建议在 Phase 1 (对外展示) 和 Phase 2 (用户服务) 之间增加一个 **"Phase 1.5: 数据迁移"**：

*   **Task 1.5.1**: 导出旧版 `txy_articles` 数据 -> 转换为 Markdown/JSON -> 导入新版数据库。
*   **Task 1.5.2**: 建立“历史归档”栏目，存放 2025 年以前的旧闻。
*   **Task 1.5.3**: 完善“服务网点”页面，复刻旧版的地图/联系方式展示。

---

**总结**:
你不必照搬旧版的所有功能（特别是论坛和评论），因为新版的目标是**提供服务**而非**闲聊**。但必须保留**历史内容资产**和**服务点检索**能力。
